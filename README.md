# 观韬团队工作平台 — 部署指南

> 前端网页 + 飞书多维表格（数据库）+ Vercel（免费托管）
> 
> 部署完成后，团队成员打开一个网址就能使用你看到的前端界面，
> 所有数据自动存在飞书多维表格里。

---

## 架构说明

```
手机浏览器 → Vercel（前端页面 + API代理）→ 飞书多维表格（数据存储）
                                          → 企业微信（提醒推送，可选）
```

- **前端**：就是你看到的那个界面，部署在 Vercel 上，免费
- **数据**：存在飞书多维表格，免费，飞书App也能直接看
- **API代理**：Vercel Serverless Functions，免费额度足够
- **提醒**：飞书自动化 + 企业微信Webhook（可选）

## 第一步：创建飞书多维表格（10分钟）

### 1.1 注册飞书

如果还没有飞书账号：
1. 访问 https://www.feishu.cn
2. 用手机号注册，创建一个团队（免费版即可）
3. 邀请团队10人加入

### 1.2 创建多维表格

1. 飞书 → 工作台 → 多维表格 → 新建
2. 命名为「观韬·团队案件管理」

### 1.3 创建「咨询知识库」表

在多维表格中创建第一张表，命名为 **咨询知识库**，字段如下：

| 字段名（必须完全一致） | 类型 | 备注 |
|----------------------|------|------|
| 问题摘要 | 文本 | 首列 |
| 法律领域 | 单选 | 选项：劳动人事、公司治理、合同纠纷、知识产权、合规风控、房产物业、财税相关、其他 |
| 我方回复 | 文本 | 长文本 |
| 回复人 | 文本 | 填写姓名 |
| 录入日期 | 日期 | 默认值设为创建时间 |

### 1.4 创建「案件总表」

新建第二张表，命名为 **案件总表**：

| 字段名（必须完全一致） | 类型 | 备注 |
|----------------------|------|------|
| 案件号 | 文本 | 首列 |
| 当事人 | 文本 | |
| 案件类型 | 单选 | 选项：诉讼、仲裁、执行、其他 |
| 案件状态 | 单选 | 选项：待立案、举证期、待开庭、庭后补充、已结案、中止/终止 |
| 主办律师 | 文本 | |
| 协办律师 | 文本 | |
| 法官/仲裁员 | 文本 | |
| 联系电话 | 文本 | |
| 审理机关 | 文本 | |
| 开庭地点 | 文本 | |
| 受案日期 | 日期 | |
| 开庭时间 | 日期 | 包含时间 |
| 举证期限 | 日期 | 公式：`DATE_ADD([受案日期], 15, "day")` |
| 庭后补充截止 | 日期 | 公式：`DATE_ADD([开庭时间], 3, "day")` |
| 备注 | 文本 | 长文本 |

### 1.5 创建「案件节点」表

新建第三张表，命名为 **案件节点**：

| 字段名（必须完全一致） | 类型 | 备注 |
|----------------------|------|------|
| 节点名称 | 文本 | 首列 |
| 节点类型 | 单选 | 选项：受案/立案、举证期限、证据交换、开庭、庭后补充材料、裁判/裁决、自定义 |
| 节点日期 | 日期 | 包含时间 |
| 节点状态 | 单选 | 选项：已完成、即将到来、待定 |
| 备注 | 文本 | |
| 排序 | 数字 | |
| 所属案件 | 关联 | 关联到「案件总表」 |

### 1.6 记下表的ID

打开多维表格，看浏览器地址栏，URL格式为：
```
https://abc.feishu.cn/base/XxxxxxYYYY?table=tblZZZZZZZZ
```

- `XxxxxxYYYY` 就是 **BITABLE_APP_TOKEN**（多维表格Token）
- `tblZZZZZZZZ` 是当前显示的表的 **TABLE_ID**

依次点击三张表，记下各自的 TABLE_ID：
- 咨询知识库的 TABLE_ID → **TABLE_CONSULTS**
- 案件总表的 TABLE_ID → **TABLE_CASES**  
- 案件节点的 TABLE_ID → **TABLE_TIMELINE**

## 第二步：创建飞书自建应用（5分钟）

需要创建一个飞书应用来获取API权限。

### 2.1 创建应用

1. 访问 https://open.feishu.cn/app
2. 登录 → 创建企业自建应用
3. 应用名称填「团队案件管理」
4. 创建后记下 **App ID** 和 **App Secret**

### 2.2 配置权限

在应用管理页面 → 权限管理 → 搜索并开通以下权限：

- `bitable:app` — 多维表格权限
- `bitable:app:readonly` — 读取多维表格
- `bitable:record` — 读写多维表格记录

### 2.3 发布应用

权限配置完成后：
1. 版本管理 → 创建版本 → 发布
2. 等待管理员审核通过（如果你是管理员，直接通过）

### 2.4 授权多维表格

回到飞书多维表格：
1. 打开「观韬·团队案件管理」
2. 右上角「...」→ 更多 → 文档应用 → 添加「团队案件管理」应用
3. 这样应用就有权限读写这个多维表格了

## 第三步：部署到 Vercel（5分钟）

### 3.1 注册 Vercel

1. 访问 https://vercel.com
2. 用 GitHub 账号登录（如果没有GitHub，先注册一个）

### 3.2 上传项目

**方式一：通过 GitHub（推荐）**

1. 在 GitHub 新建一个私有仓库，命名 `legal-platform`
2. 把下载的 `legal-app` 文件夹内容推送到仓库
3. 在 Vercel 中 Import 这个仓库

**方式二：直接拖拽部署**

1. Vercel 控制台 → New Project → 选择 "Import Third-Party Git Repository"
2. 或使用 Vercel CLI：
```bash
npm i -g vercel
cd legal-app
vercel
```

### 3.3 配置环境变量

在 Vercel 项目 → Settings → Environment Variables，添加以下变量：

| 变量名 | 值 | 来源 |
|--------|---|------|
| `FEISHU_APP_ID` | 你的 App ID | 飞书开放平台 |
| `FEISHU_APP_SECRET` | 你的 App Secret | 飞书开放平台 |
| `BITABLE_APP_TOKEN` | 多维表格Token | 浏览器URL |
| `TABLE_CONSULTS` | 咨询知识库表ID | 浏览器URL |
| `TABLE_CASES` | 案件总表ID | 浏览器URL |
| `TABLE_TIMELINE` | 案件节点表ID | 浏览器URL |
| `WECHAT_WEBHOOK_URL` | 企业微信Webhook（可选） | 企业微信群机器人 |

### 3.4 部署完成

配置好环境变量后，Vercel 会自动重新部署。

你会得到一个地址，类似：`https://legal-platform-xxxxx.vercel.app`

团队成员在手机浏览器打开这个地址就能用了。

## 第四步：团队使用

### 手机添加到主屏幕

iPhone：Safari打开网址 → 分享按钮 → 添加到主屏幕
Android：Chrome打开网址 → 菜单 → 添加到主屏幕

添加后就像一个App一样，点击图标直接打开。

### 日常操作

1. **录入咨询**：📚 知识库 → + 新增 → 选领域、填问答 → 保存
2. **新建案件**：⚖️ 案件 → + 建案 → 填信息 → 系统自动创建6个时间节点
3. **查看进度**：📋 看板 → 看全局状态、紧急节点、工作量
4. **更新节点**：点进案件 → 时间线 → 点 ✓ 标记已完成

## 常见问题

**Q: 飞书免费版够用吗？**
A: 完全够用。免费版支持多维表格、自动化、API访问，10人团队没有任何限制。

**Q: Vercel免费版够用吗？**
A: 够用。免费版每月100GB带宽、100K次Serverless调用，10人团队的使用量远远用不完。

**Q: 数据安全吗？**
A: 数据存在飞书服务器（国内），API通过HTTPS加密传输。Vercel的Serverless Function中转不存储任何数据。飞书App ID/Secret存在Vercel环境变量中，不会暴露在前端代码里。

**Q: 能自定义域名吗？**
A: 可以。在 Vercel 项目 Settings → Domains 中添加自定义域名，按提示配置DNS即可。

**Q: 企业微信提醒怎么配？**
A: 在企业微信群中添加机器人 → 复制Webhook地址 → 填入Vercel环境变量的 WECHAT_WEBHOOK_URL → 在飞书自动化中配置定时触发调用API `/api/feishu/notify/test` 测试。
